# Portfolio Content

## Workflow

1. Read [docs/CONTEXT.md](../../../docs/CONTEXT.md) and README for facts
2. **Do not invent** employers, projects, or metrics
3. Draft sections: Hero → About → Links/Projects → Contact
4. Verify all external links return 200
5. Update "last updated" date in footer

## Section guidelines

### Hero
One-sentence value proposition: who you are + what you build.

### About
Conversational tone; current focus; link to Notion for detail.

### Projects (when added)
3–5 best only. Each: Problem → Role → Approach → Result.

### Contact
LinkedIn + GitHub obvious; multiple paths to reach.

## Content sources

- README.md links
- docs/CONTEXT.md owner block
- Notion portfolio (external — link, don't duplicate)

See [reference.md](reference.md) for case study template.
