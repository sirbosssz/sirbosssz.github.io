# Portfolio content — reference

Sources: [Fonzi engineering portfolio](https://fonzi.ai/blog/portfolio-for-engineer), [ByteVerse 2026](https://www.byteverse.fyi/blog/build-portfolio-website-2026)

## Required sections

| Section | Purpose |
|---------|---------|
| Hero | Value prop in one sentence |
| About | Personality + philosophy |
| Projects | 3–5 with live demo + GitHub |
| Skills | Grouped by category |
| Contact | Email/LinkedIn/GitHub above fold |

## Case study template

```markdown
### [Project name]
- **Problem:** What needed solving
- **Role:** Your contribution
- **Approach:** Tech + trade-offs
- **Result:** Measurable outcome (if known)
- **Links:** Live demo | GitHub
```

## Common mistakes

- 20 mediocre projects instead of 3–5 strong ones
- Broken external links
- Walls of text without structure
- Resume paste with no personality
- Stale content (refresh every 3–6 months)
